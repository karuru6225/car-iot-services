# OBD-II PID リファレンス（Wikipedia 日本語訳）

出典: [Wikipedia - OBD-II PIDs](https://en.wikipedia.org/wiki/OBD-II_PIDs)（SAE J1979 / ISO 15031-5 準拠）

> **注意**: メーカーはすべてのサービス・PID をサポートする義務はない。
> Honda N-VAN 固有の対応状況は `OBD.md` および `CAN_REFERENCE.md` を参照。

---

## 変数の定義

レスポンスバイトは A（最初）、B、C、D の順で参照する。

| 変数 | 意味 |
|------|------|
| A | レスポンス data[3]（最初のデータバイト） |
| B | レスポンス data[4] |
| C | レスポンス data[5] |
| D | レスポンス data[6] |

---

## Mode 01 — 現在データ取得（Show Current Data）

リクエスト: `Byte0=0x02 Byte1=0x01 Byte2=PID`  
レスポンス: `Byte1=0x41 Byte2=PID Byte3=A Byte4=B ...`

## Mode 02 — フリーズフレームデータ

Mode 01 と同じ PID を使用する。ただし、返されるのは最後の DTC が記録された時点のスナップショット値。  
追加 PID: `0x02` → フリーズフレームを引き起こした DTC コード。

---

## Mode 03 — 保存済み DTC 一覧

DTCは1件あたり2バイト。5文字コードに変換する。

| ビット | 意味 |
|--------|------|
| bit15-14 | カテゴリ: `00`=P（パワートレイン）, `01`=C（シャシー）, `10`=B（ボディ）, `11`=U（通信） |
| bit13-12 | サブカテゴリ（0〜3） |
| bit11-0 | 故障コード番号（16進3桁） |

例: `0x0158` → P0158（O2 センサ B1S2 高電圧）

---

## Mode 04 — 診断コード消去

DTCとフリーズフレームをすべて消去し、MIL を消灯する。データなし。

---

## Mode 05 — O2 センサモニタリング（非CAN車専用）

CAN 車では Mode 06 に統合されているため通常不要。  
32 種のテストID で O2 センサの電圧閾値（リッチ↔リーン遷移）を監視する。

---

## Mode 06 — コンポーネント・システムテスト

OBD モニタの測定結果（実測値、最小/最大閾値）を返す。車種ごとに定義が異なる。

---

## Mode 07 — 保留中 DTC（未確定コード）

現ドライブサイクル中に検出された未確定の DTC を返す（消去操作なしに自動クリアされる）。

---

## Mode 08 — コンポーネント制御

ECU のアクチュエータを直接制御するためのモード。車種・ECU 依存。

---

## Mode 0A — 永久 DTC（消去不可コード）

ECU に永続保存された DTC。OBD スキャナからの消去操作では消えない。  
実際の修理完了後に自動消去される。

## 注記

### デコード式の等価表現

Wikipedia の式とコード実装では同じ意味の書き方が複数存在する：

| Wikipedia 表記 | コード等価表現 | 備考 |
|---------------|--------------|------|
| `A / 2.55` | `A * 100 / 255` | 0〜100% の変換 |
| `(A / 1.28) - 100` | `(A - 128) * 100 / 128` | 燃料トリム |
| `A / 200` | `A * 0.005` | O2 センサ電圧 |
| `(2/65536) × (256A+B)` | `(uint32_t)(A<<8|B) * 2.0 / 65536` | λ比 |

### Honda N-VAN 固有事項

- **29ビット拡張アドレッシング必須**: `0x7DF`（11bit）には無応答
- **0x05冷却水温**: 非対応 → `0x67` Sensor1 で代替
- **0x10 MAF**: 非対応 → `0x66` Sensor1 で代替
- 詳細は [OBD.md](OBD.md) 参照
