# SIM7080G (M5Stack U128) ATコマンド・運用ガイド

実機検証（M5Atom S3 + U128 + SORACOM plan-D SIM）から得た知見をまとめる。

---

## ハードウェア接続

### UART ピン配置（重要: ラベル逆注意）

```cpp
Serial2.begin(115200, SERIAL_8N1, LTE_RX_PIN, LTE_TX_PIN);
// LTE_RX_PIN = 6  (G1 ← U128 TXD)
// LTE_TX_PIN = 5  (G2 → U128 RXD)
```

U128 コネクタの **TXD/RXD ラベルは M5Atom S3 側から見た名前と逆**。
ループバックテストで疎通確認してから接続すること。

## 初期化シーケンス

### 1. モデム起動

```
modem.init()          // TinyGSM 経由。modem.restart() は PWRKEY 制御が必要なため使わない
ATE0                  // エコー無効化
AT                    // 疎通確認（リトライ 5回 推奨）
```

### 2. APN 設定（順序必須）

```
AT+CFUN=0             // デタッチ（+APP PDP: 0,DEACTIVE が返る）
delay(2000)
AT+CGDCONT=1,"IP","soracom.io"
AT+CFUN=1             // 再アタッチ
// SMS Ready を待つ（最大 15 秒）
```

**CFUN=0 → CGDCONT → CFUN=1 の順を守らないとネットワーク登録できない。**

## 証明書管理

### コマンド一覧

| コマンド | フォーマット | 説明 |
| --- | --- | --- |
| 初期化 | `AT+CFSINIT` | 操作前に必須 |
| 終了 | `AT+CFSTERM` | 操作後に必須 |
| 書込 | `AT+CFSWFILE=<dir>,<name>,<mode>,<size>,<timeout>` | `dir`: 0=/custapp/ 3=/customer/ `mode`: 0=新規 1=追記 プロンプト `DOWNLOAD` 待ち → データ送信 |
| 読込 | `AT+CFSRFILE=<dir>,<name>,<mode>,<size>,<pos>` | `mode`: 0=先頭から 1=指定位置から |
| 削除 | `AT+CFSDFILE=<dir>,<name>` | ファイルを削除 |
| サイズ確認 | `AT+CFSGFIS=<dir>,<name>` | バイト数を返す（存在確認に使用）|

### アップロード（AT+CFSWFILE）

```
AT+CFSINIT                                    // ファイルシステム初期化
AT+CFSWFILE=3,"ca.crt",0,<len>,5000          // index=3(customer), mode=0(新規)
// "DOWNLOAD" プロンプト待ち → PEM データ送信 → OK 待ち
AT+CFSWFILE=3,"client.crt",0,<len>,5000
AT+CFSWFILE=3,"client.key",0,<len>,5000
AT+CFSTERM                                    // ファイルシステム終了
```

### 存在確認

```
AT+CFSGFIS=3,"ca.crt"    // => +CFSGFIS: 1187  (バイト数が返れば OK)
```

### 注意

- SIM7080G のファイルシステムは NVRAM（電源断でも残る）
- `AT+CFUN=0` を実行するとファイルシステムがリセットされ、次回起動時にアップロードが必要になる
- mode=0 は新規作成のため、既存ファイルがある状態だと ERROR になる場合がある（CFUN=0 後のリセット後なら問題なし）

## SSL 設定

本プロジェクトで使用する設定：

```
AT+CSSLCFG="sslversion",0,3                          // コンテキスト 0、TLS 1.2
AT+CSSLCFG="CONVERT",2,"ca.crt"                      // CA LIST に変換（ERROR でも続行可）
AT+CSSLCFG="CONVERT",1,"client.crt","client.key"     // クライアント証明書＋鍵
```

### AT+CSSLCFG パラメータ一覧

| 設定項目 | コマンド例 | 説明 |
| --- | --- | --- |
| SSL バージョン | `AT+CSSLCFG="sslversion",0,3` | 1=TLS1.0 2=TLS1.1 3=TLS1.2 4=All |
| 認証モード | `AT+CSSLCFG="authmode",0,<n>` | 0=認証なし 1=サーバー認証 2=相互認証 |
| CA 証明書 | `AT+CSSLCFG="cacert",0,"ca.crt"` | FS 内のファイル名を指定 |
| クライアント証明書 | `AT+CSSLCFG="clientcert",0,"cli.crt"` | 相互認証用 |
| クライアント鍵 | `AT+CSSLCFG="clientkey",0,"cli.key"` | 相互認証用 |
| SNI | `AT+CSSLCFG="sni",0,"example.com"` | ホスト名ベースの証明書検証用 |
| 有効期限無視 | `AT+CSSLCFG="ignorentctime",0,1` | 1=RTC 未同期でも期限エラーを無視 |

## ネットワーク登録・GPRS 接続

```
// TinyGSM
modem.waitForNetwork(60000)     // ネットワーク登録待ち（最大 60 秒）

modem.gprsDisconnect()
```

## MQTT (AT+SM コマンド群)

### 設定

```
AT+SMCONF="URL","<broker>","8883"
AT+SMCONF="CLIENTID","<id>"
AT+SMCONF="KEEPTIME",60         // Keepalive 秒数
AT+SMCONF="CLEANSS",1           // クリーンセッション
AT+SMCONF="USERNAME","<user>"   // 認証が必要な場合
AT+SMCONF="PASSWORD","<pass>"   // 認証が必要な場合
AT+SMSSL=1,ca.crt,client.crt   // SSL 有効化（証明書名、クォートなし）
```

### 接続・切断

```
AT+SMDISC                       // 事前に既存接続をクリア（未接続なら ERROR が返るが無視してよい）
AT+SMCONN                       // 接続（〜30 秒タイムアウト推奨）
AT+SMSTATE?                     // => +SMSTATE: 0 (未接続) / 1 (接続中)
AT+SMDISC                       // 切断
```

### Subscribe

```
AT+SMSUB="<topic>",<QoS>        // 購読開始（QoS: 0 or 1）
// 受信時に自動通知: +SMSUB: "<topic>","<payload>"
AT+SMUNSUB="<topic>"            // 購読解除
```

### Publish (QoS 0)

```
AT+SMPUB="<topic>",<len>,0,0
// ">" プロンプト待ち → ペイロード送信（改行不要）→ OK 待ち
```

OK が返っても、TCP バッファから実際に送出されるまで数百 ms〜2 秒かかる場合がある。

## HTTP(S) (AT+SH コマンド群)

### コマンド一覧

| コマンド | 説明 |
|---|---|
| `AT+SHCONF` | HTTP(S) パラメータ設定 |
| `AT+SHSSL` | SSL 設定 |
| `AT+SHCONN` | HTTP(S) 接続 |
| `AT+SHBOD` | リクエストボディ設定（POST 用） |
| `AT+SHAHEAD` | カスタムヘッダー追加 |
| `AT+SHPARA` | クエリパラメータ設定 |
| `AT+SHCPARA` | パラメータクリア |
| `AT+SHCHEAD` | ヘッダクリア |
| `AT+SHSTATE` | 接続状態確認 |
| `AT+SHREQ` | リクエスト送信 |
| `AT+SHREAD` | レスポンス読み取り |
| `AT+SHDISC` | 接続切断 |
| `AT+HTTPTOFSRL` | ファイルを AP ファイルシステムへダウンロード |
| `AT+HTTPTOFS` | ダウンロード進捗確認 |

### AT+SHCONF — パラメータ設定

**必須設定（URL / BODYLEN / HEADERLEN の3つは必ず設定すること）**

```
AT+SHCONF="URL","https://host"      ホスト名（最大 64 バイト、スキーム含む）
AT+SHCONF="BODYLEN",<n>             レスポンスボディバッファ（0–4096）
AT+SHCONF="HEADERLEN",<n>           レスポンスヘッダバッファ（0–350）
AT+SHCONF="TIMEOUT",<n>             タイムアウト秒（30–1800、デフォルト 60）
```

**URL 長の制限への対処**

S3 の virtual-hosted-style URL はホスト名が 64 バイトを超えるため、
path-style URL（`https://s3.<region>.amazonaws.com`）を使う。

```
# NG: ホスト名 72 バイト
AT+SHCONF="URL","https://my-bucket.s3.ap-northeast-1.amazonaws.com"

# OK: ホスト名 39 バイト
AT+SHCONF="URL","https://s3.ap-northeast-1.amazonaws.com"
AT+SHREQ="/my-bucket/firmware/v1.2.0.bin",1    # パスにバケット名を含める
```

### AT+SHSSL — SSL 設定

```
AT+SHSSL=<index>,<ca証明書名>[,<クライアント証明書名>]
```

AT+SMSSL と同じインデックス規則（0=SSL無効、1-6 = CSSLCFG ctindex 0-5）。

| SHSSL index | CSSLCFG ctindex | 用途 |
|---|---|---|
| 0 | - | SSL 無効（HTTPS 非対応） |
| 1 | 0 | MQTT が占有（AT+SMSSL=1）→ **使用禁止** |
| **2** | **1** | **HTTPS 用（S3/一般 HTTPS）** |

```
AT+CSSLCFG="sslversion",1,3    # HTTPS 用 ctindex 1 に TLS 1.2 を設定
AT+SHSSL=2,"ca.crt"             # ctindex 1 を使用（MQTT と競合しない）
```

provisioning は ctindex 0（MQTT 用）のみ TLS 1.2 を設定している。
HTTPS 用の ctindex 1 は接続前に別途 `AT+CSSLCFG` で設定が必要。

### AT+SHBOD — リクエストボディ設定

```
AT+SHBOD=<len_body>,<timeout>
```

- レスポンス: `OK` ではなく **`>`** が返る（データ入力待ちプロンプト）
- データを送信し、Ctrl-Z（0x1A）で確定。`ESC` でキャンセル。
- **`CR`（0x0D）を含む文字列はそこで入力が終端される** → バイナリデータは `len_body` に実サイズを指定すること

```cpp
// 正しい実装
SerialAT.println("AT+SHBOD=<len>,10000");
// '>' プロンプトを待ってからデータ送信
SerialAT.write(binaryData, len);
SerialAT.write((uint8_t)0x1A); // Ctrl-Z で確定
// OK または ERROR を待つ
```

`len_body=0`（自動検出モード）では CR を含むバイナリが壊れるため、
バイナリ PUT には必ず実サイズを渡す。

### AT+SHREQ — リクエスト送信

```
AT+SHREQ=<url>,<type>
=> OK / ERROR
URC: +SHREQ: <type string>,<StatusCode>,<DataLen>
```

- `<url>`: パスまたは完全 URL（最大 **512 バイト**）
- `<type>`: 1=GET, 2=PUT, 3=POST, 4=PATCH, 5=HEAD

URC は非同期（OK の後に来る）。

```
AT+SHREQ="/get",1
OK

+SHREQ: "GET",200,512
```

### AT+SHREAD — レスポンス読み取り

```
AT+SHREAD=<startaddress>,<datalen>
=> OK
   +SHREAD: <data_len>
   <data>
   （datalen > 2048 の場合は複数回 +SHREAD: が来る）
   ERROR（datalen がレスポンスサイズより大きい場合）
```

- `<startaddress>`: 読み取り開始オフセット（最大 307200 バイト）
- `<datalen>`: 読み取りバイト数（最大 307200 バイト、**1回の URC で返るのは最大 2048 バイト**）
- `<data>`: バイナリ可（`\0` を含む場合は `String` ではなく `readBytes` で読む）
- NOTE: **`AT+SHREQ` の後にのみ読み取り可能**

**datalen > 2048 の場合は複数の `+SHREAD:` URC に分割して届く**。
実装を単純にするため、1回のリクエストで 2048 バイト以下に抑えること。

ストリーミングはオフセットを進めながら繰り返す。

### AT+SHSTATE — 接続状態確認

```
AT+SHSTATE?
=> +SHSTATE: <status>
   OK
```

- `0`: 切断状態
- `1`: 接続状態

### MQTT との共存

AT+SH* スタックと AT+SM* スタックは独立しており、**MQTT 接続中でも HTTPS は使用できる**。
ただし **SSL コンテキストインデックスが被ると AT+SHCONN が失敗する**。

AT+SMSSL と AT+SHSSL はインデックス規則が同じ（0=無効、1-6 = CSSLCFG ctindex 0-5）。
MQTT が `AT+SMSSL=1`（ctindex 0）を使うため、HTTPS は `AT+SHSSL=2`（ctindex 1）を使うこと。

```
# NG: ctindex 0 の奪い合い → AT+SHCONN ERROR（MQTT 接続中）
AT+SHSSL=1,"ca.crt"

# OK: ctindex が異なるため共存可能
AT+CSSLCFG="sslversion",1,3    # ctindex 1 に TLS 1.2 を設定
AT+SHSSL=2,"ca.crt"             # ctindex 1 を使用
AT+SHCONN                       # MQTT 接続中でも OK
```

### AT+HTTPTOFS — ファイルを AP ファイルシステムへダウンロード

```
AT+HTTPTOFS=<url>,<file_path>[,<timeout>[,<retrycnt>]]
=> OK
   +HTTPTOFS: <StatusCode>,<DataLen>
```

- `<url>`: ダウンロード元の完全 URL
- `<file_path>`: 保存先パス（例: `"/fota/firmware.bin"`, `"/customer/test.bin"`）
- `<timeout>`: HTTP タイムアウト秒（10–1000、デフォルト 50）
- `<retrycnt>`: リトライ回数（5–100、デフォルト 5）
- `<StatusCode>`: 200=OK、604=Stack Busy など
- `<DataLen>`: ダウンロードしたバイト数

ダウンロードはブロッキング（完了後に `+HTTPTOFS:` が返る）。1MB 超の場合は `timeout` を十分に大きくすること。

### AT+HTTPTOFSRL — ダウンロード進捗確認

```
AT+HTTPTOFSRL?
=> +HTTPTOFSRL: <status>,<curlen>,<totallen>
   OK
```

- `<status>`: 0=Idle、1=ダウンロード中
- `<curlen>`: 受信済みバイト数
- `<totallen>`: 総バイト数（不明の場合は 0）

### HTTPS PUT フロー（S3 バイナリアップロード）

```
AT+SMDISC                                    # MQTT を先に切断（接続中は SHCONN が ERROR になる）
AT+CSSLCFG="sslversion",1,3                 # TLS 1.2 を HTTPS コンテキストに明示設定
AT+SHSSL=1,"ca.crt"
AT+SHCONF="URL","https://s3.ap-northeast-1.amazonaws.com"
AT+SHCONF="BODYLEN",<len>                    # 実際のボディサイズを設定
AT+SHCONF="HEADERLEN",350
AT+SHCONN
AT+SHBOD=<len>,10000
  → >                                        # OK ではなく > プロンプトが来る
<binary data>                                # バイナリデータ送信
0x1A                                         # Ctrl-Z で確定 → OK が返る
AT+SHREQ="/bucket/key",2                     # type=2 は PUT
  → +SHREQ: "PUT",200,0
AT+SHDISC
```

S3 の presigned PUT URL ではなく公開バケットへの直接 PUT の場合、認証なしで送信できる。

### HTTPS GET フロー（OTA ダウンロード）

```
AT+SMDISC                                               # MQTT 切断
AT+CSSLCFG="sslversion",1,3
AT+SHSSL=1,"ca.crt"
AT+SHCONF="URL","https://s3.ap-northeast-1.amazonaws.com"
AT+SHCONF="BODYLEN",4096
AT+SHCONF="HEADERLEN",350
AT+SHCONN
AT+SHREQ="/bucket/firmware/v1.2.0.bin",1
  → +SHREQ: "GET",200,1048576
AT+SHREAD=0,4096        → +SHREAD: 4096 \r\n <data> OK
AT+SHREAD=4096,4096     → ...（offset を進めながら繰り返す）
AT+SHDISC
```

## Deep Sleep との組み合わせ（重要）

### 問題: SMSTATE 誤認

`esp_deep_sleep_start()` 後も SIM7080G は通電したまま。
Deep Sleep 復帰 → `setup()` で `AT+CFUN=0/1` を実行すると TCP 接続は切れるが、
**モジュール内部の SMSTATE は 1 のまま残る**。

この状態で `AT+SMPUB` を実行すると ATコマンドは OK を返すが、
実際にはブローカーにデータが届かない。

### 解決策

Deep Sleep に入る直前に必ず `AT+SMDISC` → `gprsDisconnect()` を呼ぶ。

```cpp
delay(2000);       // 最後の publish のパケット送出完了を待つ
lte.disconnect();  // AT+SMDISC + gprsDisconnect()
esp_sleep_enable_timer_wakeup(...);
esp_deep_sleep_start();
```

**`delay(2000)` が必要な理由**:
`AT+SMPUB` の OK はモジュールがコマンドを受け付けた時点で返る。
すぐに `AT+SMDISC` を呼ぶと TCP バッファに残ったパケットが送出される前に切断される。

### 復帰後の期待されるログ

```
AT+SMSTATE? => +SMSTATE: 0   // 切断済みが確認できる
AT+SMCONN   => OK             // 新規接続
[MQTT] publish OK
...
AT+SMDISC   => OK             // Sleep 前に切断
```

## 信号品質 (AT+CSQ / getSignalQuality)

| 値 | 意味 |
|----|------|
| 0〜31 | RSSI 有効値（高いほど良い） |
| 99 | 測定不能（ネットワーク登録直後などに返ることがある） |

99 が返っても GPRS 接続・MQTT 送信が成功することがある。
エラー扱いしないこと。
