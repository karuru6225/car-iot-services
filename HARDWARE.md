# ハードウェア設計メモ

## 概要

12V 車載バッテリー（サブバッテリー）から電源を取り、2系統のバッテリー電圧を監視する基板。
MCU との接続方式として以下の 2 案を検討中。

| 案 | MCU | 接続方式 |
| -- | --- | -------- |
| **案A（従来）** | M5Atom S3（外付け） | Grove ケーブル 1 本（5V / GND / SDA / SCL） |
| **案B** | ESP32-S3-MINI-1-N8（基板直付け） | カステレーションはんだ付け |

---

## システム構成

### 案A（M5Atom S3 外付け）

```text
[12V サブバッテリー]
        ↓
[本基板]
  保護回路（F1 → TVS1 → D1）
        ↓
  LM2596（12V → 5V）
        ↓
  CN4-A: GROVE コネクタ（5V + GND + SDA + SCL）
        ↓
[M5Atom S3] ← Grove ケーブル 1 本
  └── UART（GPIO5/6）→ U128（SIM7080G）← 別途 Grove 4線接続

[メインバッテリー +] → 分圧回路 → ADS1115 AIN0/AIN1
[サブバッテリー +]  → 分圧回路 → ADS1115 AIN2/AIN3
```

### 案B（ESP32-S3-MINI-1 基板直付け）

```text
[12V サブバッテリー]
        ↓
[本基板]
  保護回路（F1 → TVS1 → D1）
        ↓
  LM2596（12V → 5V）
        ↓
  ESP32-S3-MINI-1-N8（基板直付け）
    I2C ──→ ADS1115
    UART ──→ CN4-B: Grove コネクタ → U128（SIM7080G）

[メインバッテリー +] → 分圧回路 → ADS1115 AIN0/AIN1
[サブバッテリー +]  → 分圧回路 → ADS1115 AIN2/AIN3

[CN5: リレー出力] ←── FET × 3（GPIO 制御）※案B のみ
```

### リレー制御回路（案B のみ）

外部リレー（最大 3 個）を GPIO で制御するための FET ドライバ回路。

| Ref | 種別 | 仕様 | 数量 | 備考 |
| --- | ---- | ---- | ---- | ---- |
| Q1, Q2, Q3 | N-ch MOSFET | BSS138 or AO3400（SOT-23） | 3 | Vgs(th) ≤ 2V（3.3V GPIO で駆動可）、Vds ≥ 20V |
| R8, R9, R10 | チップ抵抗 | 1kΩ | 3 | ゲート抵抗（発振防止） |
| D3, D4, D5 | 小信号ダイオード | 1N4148 | 3 | フライバックダイオード（リレーコイル並列） |

```text
GPIO ──[R8 1kΩ]──→ Gate
                   Q1 (N-ch FET)
                   Drain ──→ CN5 リレー出力ピン
                   Source → GND

リレーコイル: 12V ─── コイル ─── Drain
                         │
                      [D3 逆向き]（フライバック）
```

## 分圧回路

### 定数

| 定数 | 値 |
| ---- | -- |
| R1（上側） | 680kΩ 1% |
| R2（下側） | 11kΩ 1% |
| 分圧比 | 11k / (680k + 11k) ≒ 1/62.8 |
| ADS1115 ゲイン | GAIN_EIGHT（±0.512V） |
| 理論分解能 | 約 1mV / LSB（バッテリー端換算） |

### 対応電圧範囲

- 最大: 約 32V（ADS1115 入力 0.51V）
- LiFePO4 4S: 10.0V（放電）～ 14.6V（充電）→ 0.159V ～ 0.233V
- 鉛バッテリー 12V: 10.5V ～ 14.7V → 0.167V ～ 0.234V

どちらも ±0.512V レンジ内に収まる。

## Grove コネクタ ピン配置

### CN4-A — M5Atom S3 接続（案A）

M5Atom S3 の Port A（HY2.0-4P）に直結する。

| ピン | 信号 | 備考 |
| ---- | ---- | ---- |
| 1 | GND | |
| 2 | VCC | 5V（LM2596 出力） |
| 3 | SDA | ADS1115 SDA（G38） |
| 4 | SCL | ADS1115 SCL（G39） |

Port A の VCC は 5V 出力であることを実機で確認済み。

### CN4-B — U128 接続（案B）

U128（SIM7080G）に Grove ケーブル 1 本で電源と UART を供給する。

| ピン | 信号 | 備考 |
| ---- | ---- | ---- |
| 1 | GND | |
| 2 | VCC | 5V（LM2596 出力） |
| 3 | TX | ESP32-S3 → U128 RXD |
| 4 | RX | ESP32-S3 ← U128 TXD |

## 消費電流（案B 参考値）

5V ラインのピーク電流（LTE 送信中 + BLE スキャン中の同時発生時）：

| 機器 | ピーク電流 |
| ---- | ---------- |
| ESP32-S3-MINI-1 | ~240mA |
| U128（SIM7080G） | ~250mA |
| ADS1115 | ~0.2mA（無視可） |
| **合計** | **~500mA** |

12V 入力電流（LM2596 効率 80% 想定）：約 260mA。F1 ポリヒューズ 1.5A は余裕あり。

## PCB レイアウト指針

### ゾーン分離

```text
┌──────────────────────────────────────────────────┐
│ [CN1: 12V入力]                                   │
│      ↓                                           │
│ F1 → TVS1 → D1                                  │
│                                                  │
│ ┌─[スイッチング領域]──┐    ┌─[アナログ領域]──────┐ │
│ │ LM2596             │    │ ADS1115            │ │
│ │ L1（スイッチングノード近傍）│ │ C7/C8（VDD直近）  │ │
│ │ D2                 │    │ R5/R6（プルアップ） │ │
│ │ C1 / C2            │    │                    │ │
│ └────────────────────┘    └────────────────────┘ │
│           ↑ GND は 1 点で結合                     │
│                                                  │
│ [CN2: BAT1] → R1/R2 → ADS1115 AIN0/1            │
│ [CN3: BAT2] → R3/R4 → ADS1115 AIN2/3            │
│                                                  │
│ [CN4-A: Grove → M5Atom S3]  ※案A               │
│ [ESP32-S3-MINI-1 + CN4-B: Grove → U128]  ※案B  │
└──────────────────────────────────────────────────┘
```

### 重要ルール

1. **LM2596 のスイッチングノード**（L1・D2 が接続されるノード）のパターンは短く・面積を小さく保つ。最もノイズを放射する箇所。
2. ADS1115 は LM2596 から**基板の反対側**に配置する。
3. 分圧抵抗（R1～R4）は ADS1115 の近くに置き、パターンを短くする。
4. C7/C8（デカップリング）は ADS1115 の VDD ピン直近に配置する。
5. GND プレーン（ベタ）を使う場合も、スイッチング領域とアナログ領域の GND を **1 点で接続**する（スター GND）。
6. 案B: ESP32-S3-MINI-1 の**全 GND ピンは基板 GND に接続**する（放熱・ノイズ対策）。

## ファームウェアとの対応

| 設定 | 案A（M5Atom S3） | 案B（ESP32-S3-MINI-1） |
| ---- | ---------------- | ---------------------- |
| I2C アドレス | `0x48`（ADDR → GND） | 同左 |
| SDA ピン | G38 | 要確定（ピン割り当てによる） |
| SCL ピン | G39 | 要確定（同上） |
| LTE RX ピン | GPIO6 | 要確定 |
| LTE TX ピン | GPIO5 | 要確定 |
| ゲイン | `GAIN_EIGHT`（±0.512V） | 同左 |
| 差動入力 ch1 | `readADC_Differential_0_1()` | 同左 |
| 差動入力 ch2 | `readADC_Differential_2_3()` | 同左 |

> 案B でピン番号確定後、`config.h` の `LTE_RX_PIN`, `LTE_TX_PIN` および I2C ピン定義を更新する。

## 精度向上のための推奨事項

- 分圧抵抗は **1% 精度品**を使用（5% 品は不可）
- 同一温度ドリフト品の抵抗を近傍配置してペア動作させる
- 必要に応じてソフトウェアでキャリブレーション補正を追加する
- 高精度が必要な場合は変換レートを 8SPS に設定し複数サンプル平均化を行う
